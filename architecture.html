Factory Inventory Management System

A full-stack demo application: a Vue 3 single-page app talking to a FastAPI backend that serves mock data held entirely in memory — there is no database. Data is seeded from JSON files at startup.

Data Flow example: user changes a filter

User picks a filter in FilterBar.vue — Time Period, Warehouse, Category, or Order Status. State lives in useFilters.js as module-level refs, so every view shares the same filter state.
The active view (e.g. Dashboard.vue) watches the filter refs and calls api.js, which serializes non-'all' filters into query params via axios.
FastAPI receives the request (e.g. GET /api/dashboard/summary?warehouse=W1&month=Q3-2025) and runs apply_filters() and filter_by_month() over the in-memory lists. Quarters like Q3-2025 expand to their three months.
Results are validated against Pydantic response_models (or aggregated into summary dicts) and returned as JSON.
The view stores raw responses in refs; computed properties derive chart series and totals, and Vue re-renders the SVG charts and tables.

Observations gaps worth knowing about

Client calls endpoints the server doesn't implement. api.js defines task CRUD (/api/tasks) and purchase-order calls (POST /api/purchase-orders, GET /api/purchase-orders/{backlogItemId}), but main.py has no such routes — the Pydantic models (PurchaseOrder, CreatePurchaseOrderRequest) exist, unused. TasksModal and the backlog PO flow will get 404s.
Backlog.vue exists but is not routed. main.js registers six routes; the Backlog view is not among them and the nav has no link to it.
No persistence. All data is module-level Python lists loaded at startup; any mutation vanishes on restart. purchase_orders.json is currently empty.
Demo-grade security. CORS allows all origins, there is no authentication (useAuth is hardcoded to authenticated), and the API base URL is hardcoded to localhost:8001 in api.js.
